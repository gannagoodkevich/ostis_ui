Задача: нахождения частоты использования ключевых узлов для пользователя.

Описание: агент находит все элементы, над которыми заданный пользователь произвёл действие и подсчитывает количество повторяющихся элементов. Для каждого уникального элемента формирует цепочку величин, задавая отношение количество* между элементом и количеством вхождений. Также вся цепочка заносится во множество, задаваемое вторым параметром. Повторяющиеся элементы заносятся только один раз.

Инициализация: появление в памяти вопросной конструкции, соответствующей запросу:найти наиболее релевантный ключевой узел для пользователя

Аргументы: пользователь.

Результаты:

  • если на вход подан корректный аргумент, будет выведено количество использований каждого ключевого узла для пользователя.
  • если на вход подан не корректный аргумент, будет сгенерировано сообщение о некорректном аргументе.
